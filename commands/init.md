# meta-harness-init

Initialize meta-harness for this project by running an interactive Q&A to define the project domain, evaluation criteria, ensemble preferences, and evolution settings. Writes `.meta-harness/config.yaml`.

## --general Flag (Quick Init)

If `--general` is passed as an argument, **skip ALL questions** and initialize with sensible defaults that work for any project:

- domain: `general`
- primary_metrics: `[correctness, completeness, quality]`
- ensemble mode: `auto`
- evolution: `conservative` (promote after 5 successes)

Jump directly to the **Writing the Config** section with these values, then show the **Completion Message**.

## Interactive Mode (default)

If a `[domain]` argument was provided (without `--general`), use it as the default answer for the domain question and skip asking it.

Run the following questions using `AskUserQuestion`. Ask them one at a time and wait for each answer before proceeding.

### Question 1: Project Domain

```
AskUserQuestion(
  "What is the primary domain of this project?",
  options=["backend", "frontend", "ml", "cli", "infra", "general"]
)
```

Default if skipped: `general`

### Question 2: Primary Success Metrics

```
AskUserQuestion(
  "What are the most important success metrics for tasks in this project? Select all that apply.",
  options=[
    "Test pass rate (automated tests)",
    "Build success (clean compile/lint)",
    "Performance (response time, throughput)",
    "Security (no vulnerabilities)",
    "Maintainability (code quality, readability)",
    "Accuracy (ML models, algorithms)",
    "User experience (UX, accessibility)"
  ],
  multiSelect=true
)
```

Default if skipped: `["Test pass rate", "Build success", "Maintainability"]`

### Question 3: Ensemble Mode Preference

```
AskUserQuestion(
  "When should meta-harness use ensemble mode (parallel execution of multiple harnesses)?",
  options=[
    "Auto (default: only for high-uncertainty + hard-to-verify tasks)",
    "Always (every task uses 2 harnesses — thorough but slower)",
    "Never (always single harness — faster)"
  ]
)
```

Default if skipped: `auto`

### Question 4: Evolution Aggressiveness

```
AskUserQuestion(
  "How aggressively should meta-harness evolve harnesses based on results?",
  options=[
    "Conservative (promote after 5 successes, demote after 5 failures)",
    "Moderate (promote after 3 successes, demote after 3 failures)",
    "Aggressive (promote after 2 successes, demote after 2 failures)",
    "Disabled (no automatic evolution)"
  ]
)
```

Default if skipped: `conservative`

## Writing the Config

After collecting answers, create `.meta-harness/config.yaml`:

```yaml
# meta-harness project configuration
# Generated by /meta-harness-init
version: "1.0"
generated_at: "{iso_timestamp}"

project:
  domain: "{domain}"                    # backend|frontend|ml|cli|infra|general

evaluation:
  primary_metrics:                      # from Question 2
    - correctness
    - completeness

ensemble:
  mode: "{auto|always|never}"           # from Question 5

evolution:
  enabled: true                         # false if "Disabled" chosen
  promotion_threshold: 5                # consecutive successes required (3 for moderate, 2 for aggressive)
  demotion_threshold: 5                 # consecutive failures before demotion
  target_pool: "experimental"           # evolved harnesses go to experimental first
```

Also create `.meta-harness/` directory structure if it doesn't exist:

```bash
mkdir -p .meta-harness/sessions
mkdir -p .meta-harness/evaluation-logs
mkdir -p .meta-harness/evolution-proposals
```

Also suggest adding `.meta-harness/` to the project's `.gitignore`:

```bash
grep -qxF '.meta-harness/' .gitignore 2>/dev/null || echo '.meta-harness/' >> .gitignore
```
